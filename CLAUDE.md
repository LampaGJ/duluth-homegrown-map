# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this project is

A small data-capture project for the **Duluth Homegrown Music Festival** (late April / early May, Duluth MN + Superior WI). It maintains a scraped schedule and an enriched venue sidecar, both as JSON. There is no application, no test suite, no package manifest — just a Python stdlib script and two data artifacts.

## Files

- `scrape_schedule.py` — the only code. Fetches `duluthhomegrown.org/schedule/`, parses the rendered HTML into a record-per-act list, diffs against the previous run in place, and writes `homegrown_2026_schedule.json` with a top-level `changes` block (`added` / `removed` / `moved` / `summary`) above the `schedule` array.
- `homegrown_2026_schedule.json` — output of the scraper. ~185 records: date, day_label, time_start (+ 24h variant), venue, act. Regenerated by the scraper; do not hand-edit.
- `homegrown_2026_venues.json` — hand-curated sidecar, 40 venues. Per-venue keys: address, city, state, website, phone, established, type, capacity, capacity_source, notable, neighborhood, sources, licenses. Enriched incrementally by subagents; preserve structure when editing.
- `SOURCES.md` — **read this before doing any new data-gathering work.** Documents every source used (WP/Elementor fingerprint, Duluth Legistar PDFs, WI DOR REST API at `ww2.revenue.wi.gov/WebServicesPublicWeb/rest/liquor/all`, Eventective/press for capacity), dead ends tried, and the feed-first / respectful-use norms the project operates under.

## Running the scraper

```bash
python3 scrape_schedule.py                      # writes homegrown_2026_schedule.json, diffs in place
python3 scrape_schedule.py --out new.json       # alternate output path
python3 scrape_schedule.py --prev old.json      # diff against a specific prior snapshot
```

Stdlib only (no pip deps). Idempotent — re-running against an unchanged page produces `added 0, removed 0, moved 0`.

Suggested cron:
```
0 7 * * * cd /Users/graham/Projects/homegrown && /usr/bin/python3 scrape_schedule.py >> scrape.log 2>&1
```

## Architecture notes

**Schedule scraping is a token-stream parse, not a selector tree.** The page renders the schedule twice (by-time then by-venue) as server-side Elementor HTML. The parser:

1. Strips `<script>`/`<style>` blocks first (they contain `@` which would otherwise be mistaken for the venue/act delimiter).
2. Removes zero-width characters (`\u200b\u200c\u200d\ufeff`) — Elementor injects these into some act names and they cause spurious diffs if left in.
3. Bounds to the by-time section by stopping at the second occurrence of the first day header.
4. Walks tokens matching the repeating pattern `[day header] → [time] → @ → [venue] → [act]`. Day headers (`Sun Apr 26`) and times (`7:30pm`) update state; `@` triggers record emission with the current state.
5. Year is inferred per day header: rolls forward 30 days after each festival ends so the scraper continues targeting the next edition without code changes.

**Failure mode is loud by design.** If `DAY_HEADER_RE` finds no matches, the scraper raises rather than silently producing garbage. When that happens, re-probe the feeds listed in `SOURCES.md` § 1 before rewriting the parser — Homegrown may have installed a plugin (e.g. The Events Calendar) that exposes a real feed.

**Diff identity is `(act.lower(), date)`** — the same act on a new day is a new record; the same act at a new time/venue on the same day is a `moved`. This is intentional: acts occasionally play multiple nights, and the festival sometimes shuffles slot times within a day.

## Data-gathering conventions

- **Feed-first, scrape-last.** Before scraping any new source, probe for JSON/REST/iCal/sitemap/open-data endpoints. `SOURCES.md` § 6 has the expected failure/recovery ladder.
- **First hit wins.** For capacity and license research, stop at the first credible source per venue — don't chase exhaustively. Record the source URL with every fact.
- **Never guess.** Missing data stays `null`. AI-generated search overview blurbs are not a source (they fabricate capacity numbers).
- **DBA ≠ legal entity.** Liquor licenses are held by LLCs whose names don't match the marquee (e.g. Carmody Irish Pub ← HIPPO DOWN THE STREET LLC). Use address as the stable join key when searching city records.
- **Preserve JSON structure on in-place edits.** Both data files are machine-produced; validate that the file still parses after every edit.
